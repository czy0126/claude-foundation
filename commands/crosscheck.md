# /crosscheck — 生成跨模型审阅提示

请为当前会话的所有变更生成一份审阅提示，格式化为可直接复制到另一供应商模型的 prompt。

## 1. 变更摘要
[一句话描述做了什么]

## 2. 修改文件
```
[用 git diff --stat 列出文件]
[每个文件标注：新增行数 / 删除行数]
```

## 3. 关键代码片段
[复制 2-3 个最核心的函数/类，每段不超过 30 行]
[标注对应的文件路径和行号]

## 4. 测试覆盖
[列出已有的测试文件和相关测试函数]

【修改的文件】：
- [文件1] (+X -Y 行)
- [文件2] (+X -Y 行)

【关键代码片段】：
[文件路径:行号]
```[语言]
[代码]
```

【已有测试】：
- [测试1]
- [测试2]

请逐一找出每个文件中可能存在的问题，分为严重、警告和建议三个级别。
```
