# 上下文管理规范

> 上下文窗口不是无限的——1M token 很大，但注意力会稀释。管理上下文 = 管理 Claude 的"工作记忆"质量。

---

## 上下文层级（从最持久到最短暂）

```
第 1 层：CLAUDE.md                 → 始终加载，项目级规则
第 2 层：.claude/memory/*.md       → 跨会话持久化
第 3 层：.claude/rules/*.md        → 按文件路径条件加载
第 4 层：当前任务规格              → 每次功能/任务开始时加载
第 5 层：相关源文件                → 按需读取
第 6 层：错误输出                  → 失败时反馈
第 7 层：对话历史                  → 累积，会被压缩
```

**黄金法则**：放得越高，越持久。值得跨会话保留的信息应该放在 CLAUDE.md 或 memory 文件中。

---

## 核心策略

### 策略 1：不相关任务之间 `/clear`

这是**唯一最重要的习惯**。累积的不相关上下文会严重降级性能。

```
会话 A：实现功能 X → /clear
会话 B：修复 bug Y → /clear
会话 C：重构模块 Z
```

### 策略 2：主动压缩

在 Claude 自动压缩之前主动 `/compact`：
```
/compact 保留 API 变更部分，保留测试模式，丢弃探索性搜索。
```

**压缩不会保留的内容**：
- 子目录的嵌套 CLAUDE.md（下次读文件时重新加载）
- 对话历史（被压缩摘要替代）

**压缩会保留的内容**：
- 项目根 CLAUDE.md（从磁盘重新读取）
- 系统 prompt

### 策略 3：在主要边界处全新开始

```
功能 A 完成 → 关闭会话 → 5 行摘要 → 新会话 → 加载 CLAUDE.md + memory → 功能 B
```

新会话自动获得干净的上下文 + CLAUDE.md + memory 文件。

### 策略 4：将冗长操作委托给子代理

探索阶段会读取大量文件——委托给子代理，详细输出留在它们的上下文中，只有摘要返回给你。

### 策略 5：用 PostCompact 钩子重注入关键上下文

```json
{
  "hooks": {
    "PostCompact": [
      {
        "hooks": [{
          "type": "command",
          "command": "jq -n '{hookSpecificOutput: {hookEventName: \"PostCompact\", additionalContext: \"当前分支: main。部署目标: staging。未提交文件: src/auth.py, tests/test_auth.py\"}}'"
        }]
      }
    ]
  }
}
```

## CLAUDE.md 大小管理

| 项目规模 | CLAUDE.md 建议 | 策略 |
|---------|---------------|------|
| 小型（< 50 文件） | < 200 行 | 单文件足够 |
| 中型（50-200 文件） | < 500 行 | 主 CLAUDE.md + `@imports` |
| 大型（200+ 文件） | < 300 行主文件 | 主文件作为索引 + 模块级 CLAUDE.md + `.claude/rules/` |

**模块级 CLAUDE.md 示例**：
```
project/
├── CLAUDE.md              # 高层：技术栈、命令、跨模块规则
├── src/
│   ├── core/
│   │   └── CLAUDE.md      # core/ 约定、模块职责
│   └── api/
│       └── CLAUDE.md      # API 特定模式、路由约定
```

## Memory 文件写作规范

### 应该写的

- **操作交接**：当前状态 + 关键决策 + 不要忘记的事项
- **陷阱注册表**：每个陷阱标注发现日期 + 具体修复方法
- **单主题**：一个文件只关注一个关切点，< 200 行

### 不应该写的

- 不要在 CLAUDE.md 和 memory 文件中重复相同信息
- 不要写"散文纪念碑"——代理不能从中提取具体事实
- 不要放过期信息——过期的数字比缺失更危险
- 不要写"待办事项清单"——这是 backlog，不是 memory

## 会话管理速查

| 场景 | 操作 |
|------|------|
| 任务完成，开始新任务 | `/clear` 或新会话 |
| 上下文变长（100+ 轮） | `/compact` 主动压缩 |
| 修改了 20+ 文件 | 考虑分阶段，检查点提交 |
| 两次修正失败 | `/clear` 重启——不要继续在污染上下文中迭代 |
| 需要探索大量文件 | 委托给子代理 |
| 需要快速查一个不相关的问题 | 开新会话快速查询 |
| 想知道上下文用了多少 | `/context`（如可用）或 `/doctor` |
| 配置是否有问题 | `/doctor` |
